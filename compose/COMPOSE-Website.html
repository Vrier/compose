01

AboutAn engine for the principle of compositionality

COMPOSE turns the chalkboard derivation into something students do. Every exercise is a syntactic tree; the task is to compose its meaning from the bottom up, choosing the right rule at each node until the root yields a truth condition. Inspired by the Lambda Calculator (Champollion, Tauberer & Romero), rebuilt as a browser-native engine with an authoring toolkit.

  • 01

    Trees, not multiple choice

    Each item is a bracketed syntax tree. Click a node, pick a composition rule, and the engine performs the β-reduction and checks the type. Wrong rule, wrong type — it won't combine.

  • 02

    A real λ-calculus engine

    Denotations are genuine typed λ-terms. COMPOSE infers types, applies functions, reduces, and recognises α-equivalent answers — so it grades meaning, not surface string.

  • 03

    Author your own

    A built-in editor lets you set a domain, write a lexicon, lay out trees, and toggle which rules are in play. Export to .compose.json or a self-contained assignment.

  • 04

    Teacher & student modes

    Teacher mode unlocks every rule, the editor, and answer reveals. Student builds hand out a fixed assignment with progress tracking and per-exercise rule sheets.

  • 05

    Notes built in

    Sets can carry notes — a faithful, exercise-locked companion to the chapter — shown in a side panel beside the tree.

  • 06

    Runs anywhere

    A single HTML file in the browser. No install, no account, no server. Works offline once loaded, and on phones with a dedicated mobile layout.

03

The rules of compositionEvery mechanism, switchable per exercise

COMPOSE offers the standard inventory of composition operations. An author decides which are active for a given set — so a §6 exercise sees only Function Application, while a §13 set adds intensional variants.

Basiccore · always in reach
FA

Function Application

core · default on

Apply a function of type ⟨A,B⟩ to a sister argument of type A, yielding B.

PM

Predicate Modification

core

Conjoin two ⟨e,t⟩ sisters pointwise: λx.[ A(x) ∧ B(x) ].

NN

Non-branching Node

core · default on

A node with a single child inherits its child's meaning unchanged.

Advancedmovement & intensions
PA

Predicate Abstraction

movement

λ-abstract over the index of a trace to build a derived predicate — the semantics of movement.

QR

Quantifier Raising

movement · drag-and-drop

Raise a generalized-quantifier DP above S, leaving a coindexed trace and an abstraction binder.

IFA

Intensional Function Application

§13 · intensional

Pass the evaluation world as an argument so a function can see a sister's intension, not just its extension.

Type-shifting

When a constituent has the wrong type for its sister, a type-shifter rescues the composition. COMPOSE ships fourteen, each enableable independently — sorted here from the classic Partee triangle to the event- and intension-aware closures.

Capability · §10

Mereology & plurals

The domain can be a join semilattice: individuals close under sum ⊕, ordered by a parthood relation ≤, with the pluralizing star operator *P.

That makes the whole §10 fragment derivable end to end — distributive and collective predication, and the §10.5 “full fragment” where every verb is starred so it holds of sums. Cumulative readings fall out of the relational star **R.

Sums · parthood · ★ operator · cumulativity
Capability · drag-and-drop

Quantifier Raising

A generalized-quantifier DP that can't compose in situ is raised by dragging it above its scope node. COMPOSE leaves a coindexed trace t₁ and inserts an LP binder for Predicate Abstraction to λ-bind.

That covers object quantifiers (§7.4) and inverse-scope ambiguities — give an item two target denotations to require both readings.

Traces · Predicate Abstraction · scope ambiguity

06

The file formatAuthor by hand or by export

A COMPOSE set is a plain JSON file, .compose.json. There is no "exercise type" to choose — every item is a tree to compose. You can write one in any text editor, or build it in the in-app editor and click Export.

A domain declares the constants and variables of each type; a lexicon maps surface words to λ-term denotations; rules switch composition operations and type-shifters on or off; and exercises list the trees, in bracket notation, with optional target denotations.

  • ascii λWrite L for λ, A for ∀, E for ∃, i for ι — they convert as you type.
  • [.X … ]Trees use Lambda-Calculator bracket notation; a leaf is a lexicon word, t_1 a trace, a bare number an index.
  • targetsGive two goal denotations for a scope-ambiguous item to require both readings.
  • bundlesGroup sets under another textbook in a .compose-bundle.json.
  • notesAttach companion notes, kept α-equivalent and same-typed to the set's lexicon.
frodo-runs.compose.json 
{
  "compose": 1,
  "title": "Function Application",

  "domain": {
    "constants": { "e": "f s b" },
    "variables": { "e": "x y z" }
  },

  "lexicon": [
    { "words": ["Frodo"], "denotation": "f" },
    { "words": ["runs"], "denotation": "Lx.run(x)" }
  ],

  "rules": {
    "composition": { "functionApplication": true }
  },

  "exercises": [{
    "title": "A. Intransitives",
    "items": [{
      "sentence": "Frodo runs.",
      "tree": "[.S [.DP Frodo ] [.VP runs ] ]",
      "targets": ["run(f)"]
    }]
  }]
}

CreditsAuthor & provenance

COMPOSE — the Compositional Meaning Practice and Online Semantics Engine — is built and maintained by Dr Thomas Stephen, Fellow in Linguistics at Trinity College Dublin, as part of a wider set of tools for teaching formal semantics.

It owes its design to the Lambda Calculator of Lucas Champollion, Joshua Tauberer and Maribel Romero, and its preloaded library to Elizabeth Coppock and Lucas Champollion's open-access textbook, Invitation to Formal Semantics.

The engine is free to use for teaching and learning. If it is useful in your course, a note of where it travelled is always welcome.
