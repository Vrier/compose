What students see

Every worksheet page has the same three-panel layout: the exercise list on the left, the derivation tree in the middle, and the lexicon (with a symbol reference) on the right. Students work bottom-up: click a highlighted node, pick the composition rule they think applies, type the β-reduced result, and the grader checks it against the target meaning — α-equivalent answers in any notation are accepted.

The first time a worksheet is opened, a card lists exactly the rules available in that exercise, with a one-line “how to use” for each — it shows once per worksheet, and stays available behind the ☰ Rules button.

Two supports sit alongside the exercises. Hints reveal in stages — first the strategy, then the sisters’ denotations. Notes open a reading companion beside the tree: condensed chapter notes with numbered examples, typeset trees and step-by-step denotation tables, cross-linked so each exercise scrolls the reading to its section. (Notes are written in ordinary LaTeX — expex, forest, stmaryrd — see the authoring section.)

Progress saves automatically in the student’s browser, per site area — work on /cc and the per-chapter pages shares one record, and each hosted class version keeps its own. Students can save progress to a file and restore it elsewhere (Tools menu), and a completion summary gives a screenshot-friendly per-worksheet tally. Curated pages also work offline once visited (they install as a lightweight web app).

Sending students to material

For self-study or homework from the textbooks, just link a curated page — compose.tstephen.com/cc/ch7 in the syllabus is enough; there is nothing to set up. Deep links address a single derivation (/cc/ch7/#g1.d2), handy in handouts or Moodle. For material you control — your own selection, your own worksheets, live edits mid-course — host a version from the dashboard instead: students get a stable /v/your-slug link and a printable QR handout.

Authoring worksheets

Try the full authoring surface at /editor — no account needed. A worksheet is a domain, a lexicon of λ-denotations, a rule inventory, and bracketed trees; the editor validates everything live against the real engine as you type, so a mistyped denotation or an unsolvable tree is flagged immediately.

From the sandbox you can export two things: a re-importable .compose.json (load it anywhere, including into a dashboard version later), or a self-contained student HTML — one file, locked to student view, working entirely offline, that you can email or drop into a VLE.

The fastest way to author is to adapt an existing worksheet: download any built-in from /files, edit, re-import. The JSON format is documented in FORMAT.md. Reading companions and instructor notes are written as Markdown with ordinary LaTeX for the linguistics — \ex…\xe examples (expex, linguex and gb4e all work), \begin{forest}/\Tree trees, \llbracket dog \rrbracket, $\lambda x.\,dog(x)$ — and render natively in the app.

Hosting for your class

The dashboard at /dash is where worksheets become a course. Registration needs an invite code — ask the administrator (see /about). No email is required; remember your password.

A version is your hosted worksheet collection: create one, add worksheets (import a bundle, save from the editor, or author from scratch — the editor opens with only your own material), and publish. Students reach it at compose.tstephen.com/v/your-slug — the Share dialog gives a copyable link, a QR code, and a printable A4 handout. Edits go live immediately: fix a denotation mid-week and every student sees the correction on next load, no re-distribution. Each version can carry instructor notes (same LaTeX input) that students see as a 📝 panel, and the dashboard shows an opens counter per version. Saving to the server validates every worksheet with the same engine that grades students, so a broken file cannot be published.

Files & offline

Everything students need works from the hosted pages, including offline after a first visit. For fully offline teaching, export self-contained HTML assignments from the editor — they run from a USB stick.
